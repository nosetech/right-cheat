リリースのためGitHub Releaseを使用してリリースノートを作成してください。 設定するgit tag $ARGUMENTS

以下の手順で進めてください。

1. **タグの設定**

   - masterブランチの最新コミットにタグを設定する
   - タグをリモートリポジトリにプッシュする

2. **リリースノートの作成**

   - 以下に記載する要領で、`gh release create`コマンドを使用してリリースノートを作成する
     - 前バージョンから最新コミットまでの変更内容を要約する。コミットのメッセージだけでなく、変更されているファイルの内容も確認すること。
     - 要約した変更内容をリリースノートに記載する
     - dist/RightCheat\_\*\_aarch64.dmgのファイルを添付する
     - リリースノートはドラフトとして保存する
