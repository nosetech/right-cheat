リリースのためGitHub Releaseを使用してリリースノートを作成してください。 設定するgit tag $ARGUMENTS

以下の手順で進めてください。

1. **タグの設定と確認**

   - masterブランチにチェックアウト
   - タグ `$ARGUMENTS` がmasterの最新コミットに設定されていることを確認
   - タグがリモートリポジトリにプッシュされていることを確認

2. **変更内容の分析**

   - 前バージョンのタグから現在のタグまでの変更内容を確認
   - 変更されたファイルのリストを取得: `git diff [前バージョンタグ]..$ARGUMENTS --name-only`
   - 各変更ファイルの内容を確認し、ユーザーに影響のある変更をピックアップ
   - 記載対象：UI/UX改善、バグ修正、新機能、パフォーマンス改善など
   - 記載対象外：内部構造の変更、開発ツール設定、ドキュメント内のリンク修正など

3. **リリースノートの作成**

   `gh release create` コマンドを使用してリリースノートを作成

   - **タイトル形式**: `v$ARGUMENTS`（例：v1.1.1）
   - **リリースノート本文**:
     - ## 変更内容 セクションを作成
     - ユーザー向けの変更内容をカテゴリ分け（改善、修正、新機能など）して記載
     - 変更内容は簡潔で、なぜ変更されたのかが理解できるように記載
     - ## インストール セクションを作成し、DMGファイルのダウンロードリンクを記載
   - **ファイル添付**: `dist/RightCheat_*_aarch64.dmg` を添付
   - **ドラフト保存**: `--draft` フラグで作成
