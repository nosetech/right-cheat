# RightCheat

頻繁に使用するコマンドやショートカットの一覧を表示するチートシートツールです。

## 主な機能

- コマンド,ショートカットキーの一覧表示と管理
- コマンドはクリックまたはEnterキーでクリップボードにコピー
- キーボード操作を中心とした快適な操作性
- チートシート検索機能（コンボボックス形式での選択・検索）

## 設計コンセプト

ITエンジニアの業務利用を想定した以下の特徴があります：

- 作業効率向上のため、キーボード操作を最適化
- 機密情報保護のため、インターネット通信を行わない設計

## 動作確認済みOS

Mac Tahoe 26.0.1 (ARM)

## インストール手順

### 新規インストール

1. [dmgファイルをダウンロード](https://github.com/nosetech/right-cheat/releases/download/1.2.0/RightCheat_1.2.0_aarch64.dmg)します
2. ダウンロードしたファイルを実行し、インストールします

### アップグレード（既存バージョンがある場合）

既存バージョンがインストールされている状態で新しいバージョンをインストールすると、「ファイルが壊れている」エラーが発生しやすくなります。以下の手順でアップグレードしてください：

1. RightCheat を終了します
2. `/Applications/RightCheat.app` を削除します（Finder でゴミ箱に移動）
3. [dmgファイルをダウンロード](https://github.com/nosetech/right-cheat/releases/download/1.2.0/RightCheat_1.2.0_aarch64.dmg)します
4. ターミナルを開き、以下のコマンドを実行してクォーランティン属性を削除します

```bash
xattr -cr ~/Downloads/RightCheat_*.dmg
```

5. dmgファイルを実行し、インストールします

### 初回起動時・インストール後に「ファイルが壊れている」エラーが出る場合

アプリケーションがブロックされる、または「ファイルが壊れています」というメッセージが表示される場合は、以下のいずれかの手順を実施してください：

**方法 1: xattr コマンドの使用（推奨）**

1. RightCheat を終了し、`/Applications/RightCheat.app` を削除します
2. ターミナルを開き、ダウンロードした dmg ファイルに対して以下のコマンドを実行します

```bash
xattr -cr ~/Downloads/RightCheat_*.dmg
```

3. dmgファイルを実行して再度インストールします

**方法 2:「セキュリティとプライバシー」設定の変更**

1. Appleメニューから［システム設定］→［プライバシーとセキュリティ］を開きます
2. セキュリティ欄の「お使いのMacを保護するために"RightCheat"がブロックされました」というメッセージを確認します
3. 「このまま開く」をクリックします

## ログファイル

アプリケーションのログは以下のパスに出力されます：

```plaintext
~/Library/Logs/biz.nosetech.rightcheat/RightCheat.log
```

ログファイルは最大 1MB で、古いファイルを含めて最大 3 ファイルまで保持されます。ログファイルはアプリケーション動作のトラブルシューティングに役立ちます。

## アンインストール手順

RightCheatを完全に削除するには、以下の手順を行います：

1. アプリケーションファイルを削除します
2. 必要に応じて設定ファイルも削除します：

```plaintext
/Users/[ユーザー名]/Library/Logs/biz.nosetech.rightcheat/RightCheat.log
/Users/[ユーザー名]/Library/Application Support/biz.nosetech.rightcheat/rightcheat-settings.json
```

## 初期設定

### チートシートファイルの準備

1. 以下のフォーマットでJSONファイルを作成します：

```json
[
  {
    "title": "チートシートのタイトル",
    "type": "command",
    "commandlist": [
      {
        "description": "コマンドの説明",
        "command": "コマンド"
      }
    ]
  },
  {
    "title": "チートシートのタイトル",
    "type": "shortcut",
    "commandlist": [
      {
        "description": "ショートカットの説明",
        "command": "ショートカット"
      }
    ]
  },
  {
    "title": "チートシートのタイトル",
    "type": "application",
    "commandlist": [
      {
        "description": "アプリケーション名",
        "command": "起動コマンド"
      }
    ]
  }
]
```

`type` の種類:

| type                      | 動作                                                              |
| ------------------------- | ----------------------------------------------------------------- |
| `command`（省略時も同様） | クリックまたはEnterキーでコマンドをクリップボードにコピー         |
| `shortcut`                | ショートカットキーの一覧表示（グリッド表示、コピー非対応）        |
| `application`             | クリックまたはEnterキーでコマンドを実行してアプリケーションを起動 |

チートシート JSON ファイルのサンプル

```json
[
  {
    "title": "Tauri開発",
    "type": "command",
    "commandlist": [
      {
        "description": "devモード",
        "command": "yarn tauri dev"
      },
      {
        "description": "ビルド",
        "command": "yarn tauri build"
      }
    ]
  },
  {
    "title": "tmux(プレフィックスキー入力後)",
    "type": "shortcut",
    "commandlist": [
      {
        "description": "セッションの一覧選択",
        "command": "s"
      },
      {
        "description": "新規ウィンドウ作成",
        "command": "c"
      },
      {
        "description": "ウィンドウの名前変更",
        "command": ","
      }
    ]
  },
  {
    "title": "アプリ起動",
    "type": "application",
    "commandlist": [
      {
        "description": "Slack",
        "command": "open -a Slack"
      },
      {
        "description": "Google Chrome",
        "command": "open -a \"Google Chrome\""
      }
    ]
  }
]
```

2. メニューから［RightCheat］→［Preferences］を開きます
3. Preferences画面で作成したJSONファイルを読み込みます

### ファイル管理のヒント

チートシートJSONファイルは次の場所に保存することをおすすめします：

```plaintext
~/Documents/RightCheat/
```

専用フォルダを作成することで、ファイル管理が簡単になります。

## 基本操作

### チートシート切り替え

- **マウス操作**: メインウィンドウ上部のコンボボックスから表示するチートシートを選択（テキスト入力による検索にも対応）
- **キーボード操作**:
  - 0キー: コンボボックスへフォーカス移動
  - テキスト入力でチートシートを検索

### コマンドコピー

コマンドチートシートの場合(JSONファイル内で"type": "command"を指定しているチートシート)、コマンドをクリップボードにコピーすることができます。複数行コマンドにも対応しています。

- **マウス操作**: コマンドをクリックでクリップボードにコピー
- **キーボード操作**:

  - Tab/Shift+Tab: フォーカス移動
  - Enter: コピー実行

- 1～9: 数字キーで該当のコマンドをコピー（数字順に表示されるコマンド）

#### 複数行コマンドの例

`\n` を記載することで改行します。

```json
{
  "description": "複数行コマンドの例",
  "command": "command1 && \ncommand2 && \ncommand3"
}
```

### アプリケーション起動

アプリケーションランチャーチートシートの場合（JSONファイル内で `"type": "application"` を指定しているチートシート）、コマンドを実行してアプリケーションを起動することができます。

> [!IMPORTANT]
> この機能を使用するには、macOS のアクセシビリティ設定で RightCheat を許可する必要があります。
>
> システム設定 → プライバシーとセキュリティ → アクセシビリティ を開き、RightCheat をリストに追加してトグルを ON にしてください。

- **マウス操作**: コマンドをクリックでアプリケーションを起動
- **キーボード操作**:
  - Tab/Shift+Tab: フォーカス移動
  - Enter: 起動実行
  - 1〜9: 数字キーで該当のアプリケーションを起動

> [!NOTE]
> コマンドはシェル（sh -c）経由で実行されます。macOS の `open` コマンドを使用する形式（例：`open -a Slack`）が標準的な使い方です。JSONファイルには信頼できる内容のみ記載してください。

> [!WARNING] > **コマンドの失敗はアプリ側で検知できません。** コマンドはシェル（sh -c）経由で実行されるため、存在しないアプリ名や誤ったコマンドを指定してもエラーは表示されません。コマンドが正しく動作するかどうかは、事前にターミナルで確認してください。

### ウィンドウサイズのピン留め

チートシートごとにウィンドウサイズを保存し、次回同じチートシートを選択したときに同じサイズで表示することができます。

- **ピン留め方法**: 画面右下のピンアイコンをクリック、または `p` キーを押す
- **ピン留め解除方法**: 再度ピンアイコンをクリック、または `p` キーを押す

> [!NOTE]
> ピン留め中はウィンドウのリサイズができません。ピン留めしたサイズはチートシートJSONファイルに自動保存されます。

### チートシートの編集

1. **ファイルを開く**:

   - Preferences画面のファイルパス横の編集アイコンをクリック
   - デフォルトエディタでJSONファイルが開きます

2. **変更を反映**:
   - メニュー: [View] → [CheatSheet Reload]
   - ショートカット: Cmd+R

> [!NOTE]
> 現在、アプリ内での直接編集機能はありません。外部エディタで編集後、再読み込みが必要です。

### メインウィンドウ表示制御

- メニュー: [View] → [Toggle Visible]
- ショートカット: ⌘ ^ R (初期設定)

> [!TIP]
> 現在、このショートカットキーのみPreferences画面で変更可能です。

### テーマ切り替え

Preferences画面で、アプリケーションのテーマを切り替えることができます。

- **ライトモード**: 明るい背景のテーマ
- **ダークモード**: 暗い背景のテーマ
- **システムのテーマ**: OSの設定にあわせてライトテーマまたはダークテーマに変更

選択したテーマはアプリケーション終了後も保持されます。

### 文字サイズ調整

- 文字を大きく

  - メニュー: [View] → [Increase Font Size]
  - ショートカット: ⌘ + (プラスキー。使用されているキーボードによって異なります。)

- 文字を小さく

  - メニュー: [View] → [Decrease Font Size]
  - ショートカット: ⌘ -

- 初期状態に戻す

  - メニュー: [View] → [Reset Font Size]
  - ショートカット: ⌘ 0

### 仮想デスクトップ（Space）全体での表示設定

macOS の仮想デスクトップ（Space）機能を使用している場合、Preferences画面の「Other Settings」セクションで「Visible on all workspaces」の設定を変更できます。

- **有効にした場合**: アプリケーションが全ての仮想デスクトップ（Space）で表示されます
- **無効にした場合**: アプリケーションは現在のデスクトップのみで表示されます

この設定を変更した場合、アプリケーションを再起動して設定を反映させる必要があります。
